Approach
Read existing files before writing. Don't re-read unless changed.
Thorough in reasoning, concise in output.
Skip files over 100KB unless required.
No sycophantic openers or closing fluff.
No emojis or em-dashes.
Do not guess APIs, versions, flags, commit SHAs, or package names. Verify by reading code or docs before asserting.

읽기 후 쓰기: 파일을 수정하기 전에 기존 내용을 먼저 읽어 컨텍스트를 파악합니다. 변경 사항이 발생하지 않는 한 다시 읽지 않습니다.

철저한 추론, 간결한 출력: 논리적 과정은 철저히 하되, 결과물은 핵심만 담아 간결하게 작성합니다.

효율적 파일 처리: 특별한 사유가 없는 한 100KB가 넘는 파일은 읽지 않습니다.

격식 준수: 불필요한 인사말, 아부성 멘트, 마무리의 군더더기 문구는 제외합니다.

표기 규정: 이모지나 긴 대시(em-dash)는 사용하지 않습니다.

정확성 유지: API, 버전, 플래그, 커밋 SHA, 패키지 이름 등을 추측하지 않습니다. 단언하기 전에 코드나 문서를 직접 확인하여 검증합니다.
